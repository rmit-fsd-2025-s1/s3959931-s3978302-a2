# Project Structure Documentation (my-teaching-app)

This document outlines the file and directory structure of the `my-teaching-app` project.

## Root Directory (`my-teaching-app/`)

- **`.next/`**: (Directory) Next.js build output and cache. Automatically generated.
- **`.swc/`**: (Directory) SWC compiler cache. Automatically generated.
- **`coverage/`**: (Directory) Code coverage reports generated by Jest/testing tools.
- **`node_modules/`**: (Directory) Project dependencies. Managed by npm/yarn.
- **`public/`**: (Directory) Static assets that are served directly (e.g., images, fonts, robots.txt).
- **`src/`**: (Directory) Main application source code.
- **`.eslintrc.json`**: (File) Configuration for ESLint, a code linter for identifying and reporting on patterns in JavaScript/TypeScript.
- **`.gitignore`**: (File) Specifies intentionally untracked files that Git should ignore.
- **`.prettierrc.json`**: (File) Configuration for Prettier, an opinionated code formatter.
- **`README.md`**: (File) Main project README with overview, setup, and refactoring notes.
- **`PROJECT_STRUCTURE.md`**: (File) This documentation file.
- **`eslint.config.mjs`**: (File) Alternative ESLint configuration file (ES Module format).
- **`jest.config.js`**: (File) Configuration for Jest, a JavaScript testing framework.
- **`next-env.d.ts`**: (File) TypeScript declaration file for Next.js environment variables and types.
- **`next.config.ts`**: (File) Configuration file for Next.js.
- **`package-lock.json`**: (File) Records the exact versions of dependencies used in the project. Automatically generated by npm.
- **`package.json`**: (File) Project metadata, dependencies, and scripts.
- **`postcss.config.mjs`**: (File) Configuration for PostCSS, a tool for transforming CSS with JavaScript plugins.
- **`tsconfig.json`**: (File) TypeScript compiler configuration.

## `public/` Directory

Contains static assets that are served directly from the root of the application.

- **`avatars/`**: (Directory) Contains avatar images (e.g., `avatar-1.jpg` to `avatar-12.jpg`) for tutor profiles.
- **`lecturers/`**: (Directory) Contains images for lecturer profiles (e.g., `lecturer-1.jpg` to `lecturer-4.jpg`).
- **`custom-styles.css`**: (File) Seems to be an older global stylesheet. Its current usage should be reviewed as the project primarily uses CSS Modules and Tailwind CSS. (Potentially contains styles for `circle-form`, `password-strength-meter`, etc. from `SignInPage` or other older components).
- **`letter-e.png`**: (File) Image asset.
- **`university-classroom.svg`**: (File) SVG image asset used on the homepage.

## `src/` Directory

This directory is the heart of the Next.js application, containing all the source code.

- **`app/`**: (Directory) Core of the Next.js App Router implementation. Contains page routes, layouts, and special files like `page.tsx`, `layout.tsx`, `loading.tsx`, etc.
- **`modules/`**: (Directory) Contains domain-specific or feature-based modules. Each module typically encapsulates its own components, pages (page components consumed by `app/`), utility functions, and styles.
- **`shared/`**: (Directory) Contains resources that are shared across multiple modules or the entire application, such as common UI components, global styles, type definitions, custom hooks, and context providers.
- **`tests/`**: (Directory) Contains global test setup, and potentially end-to-end tests or integration tests that span across multiple modules. Unit and component tests are often co-located with the code they test (e.g., within module directories).

### `src/app/` Directory Structure

Handles routing and layouts using the Next.js App Router.

- **`(auth)/`**: (Directory) Route group for authentication-related pages.
  - `signin/page.tsx`: Page component for the sign-in route.
  - `signup/page.tsx`: Page component for the sign-up route.
  - `layout.tsx`: Optional layout specific to the auth route group.
- **`lecturer/`**: (Directory) Routes specific to the lecturer section.
  - `page.tsx`: Page component for the main lecturer dashboard (`/lecturer`).
  - `layout.tsx`: Layout specific to the lecturer section.
  - Potentially other sub-routes for lecturers.
- **`tutor/`**: (Directory) Routes specific to the tutor section.
  - `page.tsx`: Page component for the main tutor dashboard (`/tutor`).
  - `layout.tsx`: Layout specific to the tutor section.
  - Potentially other sub-routes for tutors.
- **`.gitkeep`**: (File) Placeholder file to ensure the directory is tracked by Git even if it's empty initially or contains only generated files later.
- **`layout.tsx`**: (File) The root layout for the entire application. Wraps all pages.
- **`page.tsx`**: (File) The page component for the application's homepage (`/`).

### `src/modules/` Directory Structure

Contains domain-specific or feature-based modules. Each module typically includes:

- `components/`: React components specific to this module.
- `pages/`: Page components (usually wrapper components that import actual UI from `components/` or directly define page structure) that are imported by the route files in `src/app/`.
- `styles/`: CSS Modules or other style files for the module.
- `utils/`: Utility functions specific to this module.
- `hooks/`: Custom React hooks for the module.
- `types/`: TypeScript type definitions specific to this module (though many are in `src/shared/types/`).
- `__tests__/`: Unit/integration tests for the module's components and utilities.

**Sub-directories:**

- **`auth/`**: (Directory) Module for authentication features (sign-in, sign-up, user sessions).
- **`core/`**: (Directory) **Note:** The main `README.md` states this module was removed and its contents migrated. However, the directory still exists. It might contain remnants or was partially refactored. Original purpose was likely for shared core functionalities like contexts, base hooks, and fundamental utilities before the `shared/` directory took on this role more formally.
- **`course/`**: (Directory) Module related to course information, listings, and details.
- **`home/`**: (Directory) Module for the homepage content and specific components.
- **`lecturer/`**: (Directory) Module for lecturer-specific functionalities, dashboard components, and utilities.
- **`tutor/`**: (Directory) Module for tutor-specific functionalities, dashboard components, application processes, and utilities.

### `src/shared/` Directory Structure

Contains resources shared across the application.

- **`components/`**: (Directory) Common React components reusable in multiple modules.
  - `common/`: General-purpose common components (e.g., Modals, Toasts, Buttons, Form inputs).
  - `layout/`: Components related to the overall page layout (e.g., Header, Footer, Sidebar if they were shared).
- **`contexts/`**: (Directory) **Note:** This directory was mentioned in the main `README.md` but does not appear to exist directly under `shared/`. Contexts might be co-located within modules that provide them or not yet implemented.
- **`hooks/`**: (Directory) Custom React hooks shared across the application (e.g., `useTheme.ts`).
- **`stores/`**: (Directory) Potentially for global state management stores (e.g., Zustand, Redux slices) if implemented. (Currently empty).
- **`styles/`**: (Directory) Global stylesheets, CSS variables, theme definitions, Tailwind base/utility styles.
  - `globals.css`: Next.js global styles, often includes Tailwind directives and base styles.
  - `variables.css`: CSS custom properties (variables) for theming and consistent styling.
  - `base.css`: Base HTML element styling.
  - `responsive.css`: Styles for responsive design adjustments.
  - `overrides.css`: Global style overrides, if necessary.
  - `themes/`: Potentially for different theme files (e.g., dark theme, light theme variables).
- **`types/`**: (Directory) Global TypeScript type definitions and interfaces used in multiple modules (e.g., `user.ts`, `course.ts`, `application.ts`, `lecturer.ts`).
- **`utils/`**: (Directory) Shared utility functions that are not specific to any single module. (Currently empty).

### `src/tests/` Directory Structure

Contains test-related files and configurations.

- **`app/`**: (Directory) Tests specifically for the App Router pages and layouts (e.g., `layout.test.tsx`).
- **`mocks/`**: (Directory) Mock implementations for modules, APIs, or global objects used during testing.
- **`shared/`**: (Directory) Tests for shared components, hooks, or utilities.
- **`utils/`**: (Directory) Tests for shared utility functions (if any specific test setup is needed beyond co-location).
- **`.gitkeep`**: (File) Placeholder file.
- **`setup.ts`**: (File) Jest setup file. Used to configure or set up the testing environment before tests run (e.g., importing `jest-dom`, mocking global objects or modules like `next/navigation`, `framer-motion`).
